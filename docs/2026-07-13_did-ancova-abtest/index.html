Concept 1

ノーマル分析はなぜヘビーユーザーに引きずられるか

もっとも単純な分析は、介入後の値 Y1（例: 課金額、利用時間）をTreatment/Controlで比較する回帰式です。

Y1 = a + b・Treat + 誤差   （ノーマルな方法）

この式は、ユーザーごとの「もともとの水準」を一切考慮しません。課金額のようにヘビーユーザーの値が極端に大きい指標では、Treatment群・Control群のどちらに高額課金者がたまたま多く割り当てられたかによって、係数 b（施策効果の推定値）が大きくぶれます。

具体例

ランダム割当をしていても、サンプルサイズが有限であれば「たまたま」高額課金者がTreatment群に偏ることは起こり得ます。その場合、施策に効果がなくても b が正に見えたり、逆に本当の効果が打ち消されたりします。

判断の軸: 目的変数の分散の大部分が「個人差（ユーザー固有の水準）」に由来していないか。由来している場合、ノーマル分析はその個人差ノイズをそのまま効果推定のノイズとして引き継ぐ。

理解チェック

課金額の代わりに「ログイン有無（0/1）」を目的変数にした場合、ヘビーユーザー問題は同じ深刻さで発生するでしょうか。指標の分散の偏り方の違いから理由を説明してください。

Concept 2

DID（差分の差分析）：個人内の変化量に着目する

DIDは、介入前のベースライン測定 Y0 を使い、目的変数を「変化量」に置き換えます。

(Y1 − Y0) = a + b・Treat + 誤差   （DID）

ヘビーユーザーであっても、その人自身の Y0 を基準にするため、比較されるのは「その人の中でどれだけ変化したか」です。これにより、ユーザー固有の水準（絶対値の大きさ）そのものは差分の計算で相殺されます。

具体例

介入前に月10万円課金していたユーザーが介入後に11万円になった場合の変化量は+1万円です。介入前に月1,000円だったユーザーが介入後に2,000円になった場合も変化量は+1,000円で、絶対額は全く違いますが「変化」という土俵で比較できます。

判断の軸: ベースラインとフォローアップの間で、ユーザーの相対的な順位や水準がある程度安定している（自己相関ρが高い）ほど、DIDの単純な差分は分散を小さくしやすい。

理解チェック

DIDの式は暗黙に「Y0の係数は1」という制約を置いています（Y1-Y0はY1 = a + b・Treat + 1・Y0 + 誤差と同じ形）。この制約が現実のデータに合わない場合、何が起きると考えられますか。

Concept 3

ANCOVA（共分散分析）：係数θをデータから推定する

ANCOVAはDIDの「係数1」という制約を外し、ベースラインY0を共変量として回帰式に含め、その係数θを最小二乗法で推定します。

Y1 = a + b・Treat + θ・Y0 + 誤差   （ANCOVA）

θ=1に固定するとDIDと一致し、θ=0に固定するとノーマルな方法と一致します。ANCOVAはθをデータから推定するため、DIDともノーマルな方法とも異なる「その中間の最適な重み」を自動的に選びます。

具体例

ランダム化されたA/BテストではTreatとY0は無相関（Treatはランダム割当のため）なので、θの推定値はおおよそY0とY1の相関構造から決まります。相関が強いデータではθは1に近づき（DIDに近い挙動）、相関が弱いデータではθは0に近づきます（ノーマルな方法に近い挙動）。

判断の軸: 「DIDにすべきかノーマルにすべきか」を人手で決め打ちせず、θという1つのパラメータにデータから語らせる設計がANCOVA。

理解チェック

ある実験でθの推定値が0.05だったとします。これはベースラインと介入後の値の関係についてどのようなことを示唆しますか。この場合、DIDを使うとどうなると考えられますか。

Concept 4

なぜベースラインを使うとノイズが減るのか：条件付き分散で理解する

Y0とY1が同じ分散σ²を持ち、相関係数ρで結びついていると仮定します（ランダム化によりTreatはY0と無相関）。このとき、施策効果の推定に使われる「残差の分散」は分析デザインごとに次のようになります。

分析デザイン 効果推定に使う残差の分散 解釈
ノーマル（Y1のみ） σ² Y0を全く使わないため、個人差ノイズをそのまま抱える
DID（Y1-Y0） 2σ²(1-ρ) 差分を取ることで個人差の一部が相殺されるが、独立な誤差成分が2倍加わる
ANCOVA（最適なθ） σ²(1-ρ²) Y0で条件づけた条件付き分散。理論的に達成できる最小の残差分散

具体例：ρを変えて比較する

2(1-ρ)（DID）と1（ノーマル）を比較すると、ρ > 0.5のときDIDの方が分散が小さく（有利）、ρ < 0.5のときはノーマルの方が有利という、ちょうどρ=0.5を境にした逆転が起こります。一方(1-ρ²)（ANCOVA）はρがどんな値でも1以下であり、かつ2(1-ρ)以下（ρ=1のときのみ等しい）になります。

判断の軸: ANCOVAの残差分散は理論上ノーマル・DIDのどちらより小さいか同等になる。これは「DIDが得意な領域」と「ノーマルが得意な領域」の両方をθの推定によって自動的にカバーできることを意味する。

理解チェック

ρ=0.5のとき、ノーマルとDIDの分散はどちらも同じ値になります。実際に σ² と 2σ²(1-0.5) を計算して確認し、このときANCOVAの分散σ²(1-0.25)=0.75σ²と比べてどちらが小さいか確認してください。

Concept 5

補足：理論的背景（元記事を超えた発展知識）

元記事は「ρが高ければDID、低ければANCOVA」という実務的な目安を示していますが、これは臨床試験統計学とオンラインA/Bテスト分析の両分野で以前から研究されてきた内容と対応しています。ここでは元記事の内容と区別して、その理論的背景を補足します。

Frison & Pocock (1992)

臨床試験の反復測定データを扱った古典的な論文で、上のConcept 4と同じ比較を行い、「介入後のみの分析（POST）」と「変化量の分析（CHANGE）」の優劣が ρ=0.5 を境に入れ替わること、そして「ANCOVA」はρの値によらず常にPOST・CHANGEのどちらよりも分散が小さいか同等になることを示しました。

CUPED（Deng et al., 2013）

Microsoftのオンライン実験チームが提案した分散削減手法で、介入前のデータ（Pre-Experiment Data）を共変量として使うことで分散を (1-ρ²) 倍に削減できることを示しています。考え方の骨格はANCOVAと同じで、ρが高いほどサンプルサイズを削減できる（同じ検出力をより少ないユーザー数で達成できる）という実務上のメリットに直結します。

ポイント: 元記事の「使い分け」は実務上の直感として妥当ですが、理論的にはANCOVA（＝適切な共変量調整）を選んでおけば、DIDが有利な状況・ノーマルが有利な状況のどちらでも損をしない、というのがより厳密な結論です。ただしρがデータから安定して推定できるサンプルサイズがある前提です。

Concept 6

Pythonでの実装例

いずれの分析もOLS回帰として実装できます。statsmodelsを使うと3つのデザインを同じインタフェースで比較できます。

import statsmodels.formula.api as smf

# df の列: y0（ベースライン）, y1（フォローアップ）, treat（0/1）

# ノーマル
model_normal = smf.ols("y1 ~ treat", data=df).fit()

# DID（変化量を目的変数にする）
df["diff"] = df["y1"] - df["y0"]
model_did = smf.ols("diff ~ treat", data=df).fit()

# ANCOVA（y0を共変量として含める）
model_ancova = smf.ols("y1 ~ treat + y0", data=df).fit()

for name, m in [("normal", model_normal), ("did", model_did), ("ancova", model_ancova)]:
    print(name, "coef(treat) =", m.params["treat"], "se =", m.bse["treat"])

実装時の注意点

model_ancova.params["y0"] が本文でいうθの推定値です。この値が1に近ければDIDに近い挙動、0に近ければノーマルに近い挙動をしていることが確認できます。また、Treatmentとの交互作用（treat:y0）を追加すると、施策効果がベースライン水準によって異なるかどうかも検証できます。

理解チェック

上のコードでmodel_ancova.bse["treat"]（ANCOVAの標準誤差）がmodel_normal.bse["treat"]（ノーマルの標準誤差）より小さくなるのは、どのようなy0とy1の関係があるときだと考えられますか。

使い分け早見表

観点 ノーマル DID ANCOVA
回帰式 Y1 = a + b・Treat Y1-Y0 = a + b・Treat Y1 = a + b・Treat + θ・Y0
ベースラインの扱い 使わない 係数1固定で差し引く 係数θをデータから推定
ヘビーユーザー耐性 低い ρが高いときに強い ρの値によらず安定
実装コスト 最小 小（差分列を作るだけ） 小（共変量を1列足すだけ）
必要な前提 特になし ベースライン測定が取得できる ベースライン測定が取得でき、θ推定に十分なサンプルサイズがある

適用判断の手順

  1. 目的変数（課金額・利用時間など）の分布を確認し、ヘビーユーザーによる歪みが大きいかを見る。
  2. 介入前のベースライン測定（Y0）が取得可能かを確認する。取得できないならノーマルな方法しか選べない。
  3. 過去データからY0とY1相当の指標の相関ρを概算する。
  4. サンプルサイズに余裕があるなら、まずANCOVAを既定の選択肢として検討する（θを推定できるため）。
  5. サンプルサイズが小さくθの推定が不安定になりそうな場合は、ρの高さに応じてDID／ノーマルのどちらかを簡易的に選ぶ。
  6. 実装後はθの推定値と標準誤差を確認し、想定と乖離していないかをチェックする。

まとめ・次のステップ

ヘビーユーザーの影響でA/Bテストの効果判定がぶれる問題は、目的変数の個人差ノイズをどれだけ取り除けるかという分散の問題として理解できます。ノーマルな分析はベースラインを使わずこのノイズをそのまま抱え、DIDはベースラインとの差分を取ることで自己相関が高い場合にノイズを減らし、ANCOVAはベースラインの係数θをデータから推定することで、DIDが得意な領域・ノーマルが得意な領域の両方を理論的にカバーします。

次のステップとしては、自分の業務データでY0とY1の相関ρを実際に計算してみること、そしてPythonの実装例を使ってノーマル・DID・ANCOVAの標準誤差を比較し、どの程度分散削減の効果があるかを体感することをおすすめします。余裕があれば、Treatmentとベースラインの交互作用項を加えたモデルにも挑戦してみてください。
